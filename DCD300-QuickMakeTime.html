For this week's Quick Make, you will work with a tool called Twine -- a freely available platform for building hypertexts through a visual layout logic. It is very easy to get started, but contains LOTS of possibilities for more complex coding or programming or otherwise using the computational / digital space. (We will return to Twine for a few other Quick Makes this semester, because of this).


PROMPT:  This week, your quick-make will explore additional complexity in Twine -- this time focusing on creating narrative effects through real-time limitations and constraints. This quick-make is inspired by Queers in Love at the End of the World, where we are forced to experience the game in 10 seconds of real time, a computational effect that prompts a frenzied, realist experience of an end of the world count-down

For this quick-make, imagine a scenario that could appear in a narrative, but  that can be (and in real life, would be), constrained by time, a build a short Twine text featuring this constraint. Your text can be about anything, but it must include the following:

  • at least 3 passages
  • at least 1 passage bound by time* where 1) the time constraint makes sense to the scenario; 2) the passage is designed such that the time constraint is felt by the reader/player

*You will need to use the Harlowe Guide and / or the WYSIWYG to find the code to do this

During this (and every) Quick-make, I encourage you to also explore writerly storytelling techniques -- think about story instances where a reader might reasonably need to write something, for instance. As well, if you are familiar with Twine (as I imagine most of you are), I encourage you to try out a technique you have not used before (beyond the requirements for this quick make).

See the full assignment details for exporting and submitting instructions.

SUBMISSION: Remember that if you are working with the online version of Twine, you CANNOT JUST SHARE THE LINK! When you are done, select PUBLISH TO FILE from the bottom left menu. This will create an HTML file and THAT is what you will submit here.
